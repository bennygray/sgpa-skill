# 全局回归执行规范（Regression Protocol）

> **适用**：/SGE Step 3 全局回归
> **触发**：每次编码变更后必须执行

---

## 执行标准

### 1. 运行命令

```bash
npm run test:regression
```

### 2. 通过标准

- **退出码必须为 0**
- 所有验证项 PASS
- 无 FAIL 项

### 3. 失败处理流程

```
回归失败
    ├── 分析失败原因
    │   ├── 新代码引入的 Bug → 修复新代码 → 重新运行
    │   ├── 新代码改变了预期行为（by design）→ 更新回归脚本 + 记录理由
    │   └── 回归脚本本身有 Bug → 修复脚本 + 记录
    └── 重新运行 → PASS → 继续
```

### 4. 回归脚本修改规则

如需修改回归脚本（`scripts/regression-all.ts`）：

1. 必须说明修改理由（为什么旧断言不再适用）
2. 新断言必须比旧断言更精确（不允许放宽标准）
3. 修改记录追加到回归脚本文件头部注释

---

## 回归覆盖范围

### 当前回归项

| 系统 | 验证脚本 | 验证内容 |
|------|---------|---------|
| 核心引擎 | 内置冒烟测试 | 默认 state → 10 ticks → 灵气增长 |
| 存档迁移 | 内置迁移链测试 | v1→v2→v3 字段完整性 |
| Phase B 灵田+炼丹 | `verify-phaseB-alpha.ts` | 灵田产出、炼丹成功率、丹药效果 |
| Phase C 突破+丹药 | `verify-phaseC.ts` | 突破概率、丹药使用、Buff 计算 |

### 新增系统的回归要求

每个新系统上线后，必须向 `regression-all.ts` 新增对应验证逻辑：

1. 核心功能冒烟测试（最基本的 happy path）
2. 与现有系统的交互验证（如新系统产出被其他系统消耗）
3. 存档迁移验证（旧存档 + 新字段默认值）
