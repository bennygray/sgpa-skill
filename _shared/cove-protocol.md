# CoVe（Chain of Verification）证据验证协议

> **版本**：v1.0 | **引用方**：`_shared/review-protocol.md` L2 层
> **核心原则**：独立性 — 回答验证问题时**不参考原结论**

---

## 四步执行法

### Step 1：还原原始产出

提取被审查的具体内容段落（原文引用，不做改写）。

### Step 2：生成验证问题

针对 WARN/BLOCK 发现，生成 2-3 个**可独立验证**的问题。

问题要求：
- 每个问题必须可以通过**代码搜索、文件读取、或逻辑推导**独立回答
- 禁止生成需要「重新审查全文」才能回答的问题
- 优先生成代码级验证问题（如「该字段是否在 X 文件中被使用？」）

示例：

| 发现 | 验证问题 |
|------|---------|
| "aura 字段有 3 个写者，存在竞态" | Q1: aura 的写入是否在同一个 tick 的不同阶段？Q2: 是否存在先读后写的依赖关系？ |
| "资源 X 无 Sink，有通胀风险" | Q1: 资源 X 在代码中是否有任何消耗逻辑？Q2: 设计文档中是否规划了消耗机制？ |

### Step 3：独立回答

> ⚠️ **关键规则**：回答每个验证问题时，**禁止参考原审查结论**。
> 必须从代码/文档/数据出发，独立得出答案。

执行方式：
1. 使用 `grep_search` / `view_file` 搜索相关代码
2. 基于搜索结果回答每个问题
3. 记录证据来源（文件名 + 行号）

### Step 4：对比修正

将 Step 3 的独立答案与原审查结论对比：

| 对比结果 | 动作 |
|---------|------|
| 独立答案与原结论**一致** | 维持原判定（WARN 或 BLOCK） |
| 独立答案与原结论**矛盾** | 升级判定（WARN→BLOCK）或修正发现描述 |
| 独立答案显示**无证据支撑原结论** | **移除该发现**，降为 PASS |

---

## 输出格式

```markdown
### CoVe 验证 — [发现编号]

**原结论**：[WARN/BLOCK] [描述]

**验证问题**：
1. [Q1]
2. [Q2]

**独立答案**：
1. [A1]（证据：`file.ts` L42）
2. [A2]（证据：代码搜索无结果）

**对比结果**：[一致/矛盾/无证据]
**最终判定**：[维持 WARN / 升级 BLOCK / 移除（降为 PASS）]
```
